The product has enough substance to build a credible pitch deck today. The demo mode alone carries the product story through a live walkthrough.

## What You Can Show

### Product (live demo-able)
- **Web interface** — full Quasar PWA with VM management, AI agent, real-time WebSocket updates
- **Terminal UI** — Ink-based TUI with identical feature set, targeting CLI-native NixOS admins
- **4-tier model working** — Free / Solo / Team / FabricK with tier-switcher toolbar in demo mode
- **Upgrade nag flow** — Free users see conversion prompts when hitting Weaver Solo+ features (both web + TUI)
- **AI agent** — BYOK model with streaming responses, mock mode for demos
- **Version switcher (complete)** — interactive roadmap built into the demo; each version (v1.0–v3.3) is a fully mocked, dated product state with version-gated UI sections. Replaces the static roadmap slide in investor presentations. Additive model: each version shows the complete product at that milestone.
- **Version milestone modals** — 9 architectural milestones auto-fire when stepping versions. Each shows "What shipped," "Why it matters" (bullet points), tier progression bar, and tier upgrade callouts. Tells the foundation-to-finish story without manual narration.
- **Demo flow** — blank canvas → first-run wizard (investor sees real onboarding) → VMs appear progressively → v1.0 milestone modal. Tier buttons jump to milestone versions with modal. "Private Demo" label resets everything.
- **Compliance page** — live in the product, mapped to HIPAA/SOC 2/PCI-DSS/NIST/CMMC with tier qualification per control. Screenshot-ready for Slide 3.

### Monetization Story
- **Clear tier boundaries** — Free (VM management, distro catalog, AI BYOK, topology read-only), Weaver Solo ($149/yr FM / $249/yr: Live Provisioning, managed bridges, DNS, firewall, health probes), Weaver Team ($129/user/yr FM / $199/user/yr: peer federation, Smart Bridges, auto-TLS), FabricK ($1,299/yr FM / $2,000/yr per node: multi-user, RBAC, audit, quotas, multi-host fleet)
- **K8s competitive positioning** — dedicated Slide 9 ("The Microservices Opportunity") + standalone sales doc (KUBERNETES-COMPETITIVE-POSITIONING.md) + per-industry K8s callouts across all 13 vertical docs. Bridge convergence: 3 K8s components → 1 bridge. Cost comparison: $20K/yr replaces $500K+/yr.
- **Glasswing AI threat narrative** — Project Glasswing validation woven into IT-FOCUS and WEAVER-VALUE-PROPOSITION. Hardware isolation + hypervisor diversity + NixOS deterministic patching as the response to AI-discovered zero-days at scale.
- **Conversion funnel built into product** — Free distro browsing = window shopping, Live Provisioning behind Weaver Solo+ paywall = convenience upgrade trigger
- **Observer as TAM expander (Decisions #101, #102, #129)** — `weaver-observer` (Rust, static binary, any Linux) lets FabricK observe non-NixOS hosts. 5× free Observer headroom per Managed node. Every Observed host has a "Convert to Managed" CTA. TAM expands from ~466 NixOS companies to any Linux fleet.
- **Self-serve upgrade path** — nag components show feature previews + pricing URL
- **Channel partner strategy** (Decision #38) — NixOS consulting firms (Tweag, Numtide) as FabricK front door
- **Design Partner program** — 3–5 prominent NixOS companies, $30k/yr Partner tier, marketing rights

### Technical Differentiators
- **NixOS-native** — first-class NixOS module deployment (not containerized), declarative config
- **Dual interface** — web + TUI from same backend, same tier gating, same API
- **Hypervisor-agnostic** — qemu, cloud-hypervisor, crosvm, kvmtool, firecracker support
- **1,522 tests (unit + backend + TUI + E2E)** — production-quality test coverage
- **24 automated compliance auditors** — static analysis pipeline enforces tier parity, doc freshness, license compliance, SAST, bundle size, and documentation-to-code consistency on every push
- **Security audit completed** — red team audit with 21 findings, all dispositioned; 4 hardening fixes applied; SAST auditor added to CI
- **Observer agent (v2.3+)** — `weaver-observer` Rust binary runs on any Linux, bringing non-NixOS hosts into the FabricK fleet map with read-only visibility. Eliminates the "NixOS-only" TAM constraint.

## Deck Format

The pitch deck ([PITCH-DECK.md](PITCH-DECK.md)) is a 12-slide markdown script: Problem → Solution → Product → Market → Business Model → Roadmap → Smart Bridges → FabricK → Microservices Opportunity → Competitive Positioning → Traction → The Ask. Plus appendices (verticals, success programs, competitive scorecard, live demo). Content-first, version-controlled, easy to iterate. Paste into Keynote/Google Slides/Figma when visual polish is needed.

## Audience-Specific Angles

**For VCs/angels:** Lead with Observer TAM expansion — FabricK isn't limited to NixOS, it observes any Linux fleet. Then market size (cloud-native/edge compute), NixOS growth trajectory, open-core model proven by GitLab/Grafana.

**For NixOS community:** Lead with the problem (no GUI for microvm.nix), open source commitment (free tier is real, not crippleware), TUI-first respect for CLI users.

**For FabricK prospects:** Lead with Observer — "see your entire fleet today, convert incrementally." Then audit log, RBAC, quotas/ACL, and the fact that it's already built (not vaporware). NixOS = reproducible deployments = compliance story. Industry-specific sales docs available for healthcare, defense, financial services, and pharma — use the relevant one to map product features to the prospect's regulatory framework.
