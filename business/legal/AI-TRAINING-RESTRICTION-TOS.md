# AI Training Restriction — Terms of Service Clause (Draft)

**Status:** Draft — requires attorney review before publication
**Date:** 2026-04-02
**Context:** Supplements the LICENSE file. AI Training Restriction applies to all tiers: Weaver Free (AGPL-3.0 + Commons Clause) and Solo/Team/Fabrick (BSL-1.1). See Decision #137.

## Purpose

The LICENSE file's AI Training Restriction prohibits using Weaver's source code to train AI/ML models. This ToS clause reinforces that restriction for users who interact with Weaver via the hosted demo, documentation site, or API — contexts where the LICENSE file may not be prominently displayed.

## Draft Clause

> **AI Training Restriction.** You may not use any portion of the Software, its source code, documentation, API responses, or output data to train, fine-tune, or otherwise develop artificial intelligence or machine learning models. This restriction applies regardless of whether the Software is accessed via source code, compiled binary, hosted service, API, or documentation portal. This restriction does not apply to: (a) using Weaver's built-in AI features (AI diagnostics, BYOK agent) as intended; (b) Weaver's own development process (Forge agents operating on the codebase under whizBANG Developers authorization).

## Placement

This clause should appear in:
1. Terms of Service on the product website
2. Demo site footer/banner
3. API documentation header
4. Contributor License Agreement (CLA) if adopted

## Legal Review Notes

- Attorney should confirm this clause is enforceable in the jurisdictions where Weaver is sold
- Consider whether this needs a separate click-through acceptance or if inclusion in ToS is sufficient
- Review interaction with AGPL-3.0 source code access rights (AGPL requires source availability but does not require permitting AI training use of that source)
