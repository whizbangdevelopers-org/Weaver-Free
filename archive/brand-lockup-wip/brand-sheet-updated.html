Bomb Placement Rules
WHERE THE BOMB APPEARS:
1. App footer — "a whizBANG! Developers LLC product" badge
2. Login page — company attribution below the form
3. Help page about section — full company info card
4. PWA splash / loading — understated below product mark
5. GitHub org avatar — the org-level profile image
6. README badges — shield-style repo badges
7. whizbangdevelopers.com — as the site favicon and logo
WHERE THE BOMB DOES NOT APPEAR:
× App header / toolbar (product spark territory)
× Navigation items
× VM cards or dashboard content
× PWA icon (that's the product spark)
× Anywhere it would compete with the product mark
FUSE SPARK COLOR RULE:
The bomb's fuse spark matches the current product's color.
• MicroVM = amber fuse spark
• Qepton = blue fuse spark (when backfilled)
• Gantry = TBD fuse spark
The bomb body + green ring stay constant. Only the fuse spark changes per product.
