# Weaver — Master Plan

**Last updated:** 2026-04-04 (EA→FM rename #136, fleet bridge + AI inference decisions #113–#119, à la carte model retired #96)

## Project Vision

A lightweight, mobile-first VM control plane that manages fleets across multiple hosts. Kubernetes-inspired concepts without Kubernetes complexity. NixOS-native but platform-agnostic.

**Tagline:** "Manage your entire VM fleet from your pocket."

## 3-Tier + Integrated Extensions Model (decided 2026-03-03, extended 2026-03-18/21)

**Demo tier:** All features run against mock data. A toolbar toggle lets visitors switch between Weaver Free / Weaver Solo / Weaver Team / Fabrick views — locking features in real-time using the same gating middleware as production.

**Tier positioning:**
- **Weaver Free** (32GB) = "Use the VMs you already have, from anywhere" — Weaver page, manage existing VMs, native mobile app (Android GA · iOS v1.3.x), Tailscale remote access, adoption hook. (Decision #87)
- **Weaver Solo** (128GB, $249/yr standard · $149/yr FM) = "Live Provisioning on your machine" — create VMs without `nixos-rebuild switch`, distros, bridges, push notifications, mobile deep links, Integrated Extensions. Single operator. FM slots: first 100, cap v1.2. Ships v1.0+. (Decision #76, #87)
- **Weaver Team** (128GB/host, $199/user/yr standard · $129/user/yr FM) = "Live Provisioning + Smart Bridges + manage your whole team's infrastructure" — everything Solo has, plus Weaver AI (Smart Bridges basic: automated blue/green, health routing), full management of up to 2 peer Weaver hosts (start/stop/restart/provision/delete). Ships v2.2+. (Decision #76, #87, #130, #142)
- **FabricK** (256GB/node: $2,000/yr first node · $1,250/2-4 · $1,000/5-9 · $750/10+, standard; $1,299/yr FM. 512GB/node: $2,500/yr standard · $1,750/yr FM) = "Full fleet governance + FabricK AI (Smart Bridges full + inference) + all extensions included" — per-VM RBAC, quotas, bulk ops, audit log, policy routing, FabricK fleet control, GPU scheduling, fleet inference routing. RAM ceiling is a pricing variant, not a separate tier. FM slots: first 20 (256GB) / first 10 (512GB), cap v2.2. (Decision #63, #64, #87, #142)
- **Contract** (512GB+ in 512GB blocks, sliding scale) = AI/HPC and large cloud deployments. Annual contract. (Decision #65)

**Insurance principle (decided 2026-03-03):** Free tier = zero security features, zero liability exposure. No scanning, no display, no implied security posture for unpaid users. Features behind paywall = insurable claims backed by audit trails.

```
┌────────────────────────────────────────────────────────────┐
│  v3.0: Advanced Clustering                                  │
│  HA, live migration, shared storage, resource scheduling    │
│                                                              │
│  + Edge Management (NixOS devices only)                      │
│    microvm-anywhere pattern: same makeMicroVM factory +      │
│    templates deploy to edge nodes over SSH                   │
│    Fleet orchestration via fleet-deploy.sh                   │
│    Impermanence: boot-clean resilience for edge              │
└────────────────────────────────────────────────────────────┘
```

**Edge scope:** NixOS nodes only. Immutable, declarative edge deployments with zero drift and atomic rollbacks. Not competing with K3s/SNO for generic container orchestration — managing NixOS microVMs at the edge. Generic ARM/IoT/embedded is out of scope.

**Implementation path:** The [microvm-anywhere template patterns](research/microvm-anywhere-nix-templates.md) are the foundation. The same `makeMicroVM` factory function and composable NixOS module templates (base, web, database, worker) that provision datacenter VMs can deploy/reprovision NixOS microVMs on remote edge nodes over SSH via `fleet-deploy.sh` and nixos-anywhere. Impermanence provides boot-clean resilience for unreliable edge environments (power loss, tampering). `genList` scales fleets of identical edge workers with one number change.

**Target segments:** Manufacturing floors (23% of $22B edge software market), video/camera nodes (29%), retail (hundreds of identical nodes), IoT automation (27%).

**Dashboard integrations:**
- Display fleet manifest history (what was deployed when, to which edge nodes)
- Trigger fleet deployments via API (provision/reprovision edge nodes)
- Fleet status derived from edge agent heartbeats
- Edge node health monitoring (connectivity, drift detection)

**Decision:** See Decision #35.

| 115 | Fleet bridge state management — hub authority, per-host DR, cold start policy | Fleet bridge routing state is dynamic (AI adjusts weights, blue/green mid-flight, auto-scaling adds/removes endpoints). The hosts ARE the disaster recovery mechanism. **Cold start policy (initial routing):** fleet bridge routing policy is defined in the Fabrick hub's data model (not NixOS per-host config, since fleet bridges are a hub-level concept). Cold start policy is the baseline applied when the hub starts fresh. (2) Hub rebuilds → reads cold start policy from its database backup or imported manifest. (3) Hub connects to hosts → collects last-known weights from all connected hosts. (4) Hub reconciles: last-known state from hosts is more recent than cold start baseline → hub adopts the fleet's current state. (5) Hub resumes authority → pushes any corrections back to hosts. **Network partition:** partitioned hosts keep serving on last-known weights. When partition heals, hub collects and reconciles. | 119 | Snapshot-based model provisioning — seconds-level inference deployment from Shed | Inference VMs are expensive to provision from scratch (3–10 minutes for large models: OS boot, driver load, model weights pull, model load into VRAM).
