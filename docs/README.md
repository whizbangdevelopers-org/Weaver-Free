# Documentation

This directory contains project documentation organized by category.

## Structure

| Directory | Purpose |
| --------- | ------- |
| `planning/` | Project planning, roadmaps, release plans |
| `workflows/` | Development processes, CI/CD patterns |
| `setup/` | Environment setup, disaster recovery |
| `security/` | Security audits, procedures |
| `testing/` | Platform-specific testing guides |
| `platforms/` | Platform-specific build guides |

## Root Documentation

| File | Purpose |
| ---- | ------- |
| `README.md` | Project overview |
| `CONTRIBUTING.md` | Contribution guide |
| `SECURITY.md` | Security policy |
| `TESTING.md` | Testing overview |
| `CHANGELOG.md` | Version history |
| `CLAUDE.md` | AI assistant context |

## Adding Documentation

1. Choose the appropriate category directory
2. Use UPPERCASE filenames (e.g., `FEATURE-NAME.md`)
3. Update this README if adding new categories
